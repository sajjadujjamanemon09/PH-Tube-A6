1. Discuss the scope of 'var', 'let', and 'const'.

- var scoped : This scoped available through the whole window otherwise only with the function where they declared.Var declare globally scoped.They can be accessed before declared but their value will be undefined.

- let scoped : This scoped available with curly braces where its declared.This is a block scoped declarations.'let' declarations are not hoisted.Its temporal dead zone from the start of the block until they are declared and initialized.Its mean that the result will ReferenceError when any attempts to access them before they are declared

- const scoped : This scoped are also a block scoped but it can't be reassigned or re-declared.'const' declarations are also not hoisted.Its also temporal dead zone from the start of the block until they are declared and initialized.Its mean that the result will ReferenceError when any attempts to access them before they are declared.
'let' & 'const' declarations are mostly same.

2. Tell us the use cases of 'null' and 'undefined'.

Null and undefined are both used to represent an empty value in JavaScript, but they have different meanings and use cases.

- null :

  1. 'null' is usually assigned by a programmer to indicate that a variable intentionally has no value.
  2. It is an object type.
  3. When you want to indicate that a variable is expected to hold an object or value later in the program but doesn't yet have a meaningful value, you can initialize it with null.

- undefined :

  1. undefined means that a variable has not been assigned any value yet.
  2. It is a type of its own.
  3. If you try to access an object property that doesn't exist, it will return undefined.
